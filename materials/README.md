# Materials

Drop anything here that you've already done before starting this project. When you run `/office-hours`, Claude will review these files to understand your context — so you don't have to explain everything from scratch.

## What to put here

- **Research** — market research, competitor analysis, user interviews, survey results
- **Existing code** — prototypes, proof of concept, code snippets you want to reuse or reference
- **Design** — wireframes, mockups, Figma exports, screenshots of UI inspiration
- **Docs** — pitch decks, business plans, feature specs, meeting notes
- **Data** — sample datasets, API docs from third-party services you plan to integrate
- **References** — articles, tutorials, architecture diagrams, tech comparisons

## Supported formats

Any file format works. Claude can read text files, images, PDFs, and code. For binary files (Figma, Sketch), export to PNG/PDF first.

## How it's used

During `/office-hours`, Claude scans this folder to:
- Understand what you've already built or researched
- Avoid asking questions you've already answered
- Incorporate your existing work into the PRD and product plan
- Identify reusable code or patterns for the sprint backlog

You can also reference these files later during development — just point Claude to `materials/[filename]`.

## Organization (optional)

You can organize into subfolders if you have a lot of files:

```
materials/
├── research/          Market research, competitor analysis
├── code/              Existing prototypes, snippets
├── design/            Wireframes, mockups, screenshots
└── docs/              Specs, notes, pitch decks
```

Or just dump everything flat — Claude will figure it out.
